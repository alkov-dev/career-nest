# 1. Копируем пример переменных окружения и заполняем их своими данными
cp .env.example .env  
# (Если файла .env.example нет, создайте .env вручную на основе того, что я давал в прошлом README)

# 2. Поднимаем инфраструктуру (Postgres, Redis, MinIO)
docker compose up -d

# 3. Ждем 5-10 секунд, пока БД и Redis полностью инициализируются
sleep 10 

# 4. Устанавливаем зависимости Node.js
npm install

# 5. Генерируем Prisma Client (должно быть ДО миграций)
npx prisma generate

# 6. Применяем миграции базы данных
npm run db:migrate:up 

# 7. Запускаем приложение в режиме разработки
npm run start:dev

Необходимо для загрузки иконки компании при регистрации в дальнейшем для хранения pdf
🛠 Пошаговая настройка MinIO (Первый запуск) 
После запуска контейнеров вам нужно подготовить хранилище для файлов:
Откройте браузер и перейдите по адресу: http://localhost:9001
Войдите с учетными данными: minioadmin / minioadmin123.
В левом меню перейдите в Buckets (или нажмите Create Bucket на главной).
Создайте бакет (например, с именем career-uploads).
(Опционально) Если вы хотите, чтобы файлы (например, аватарки) были доступны по публичным ссылкам, зайдите в настройки созданного бакета -> вкладка Access Policy и измените правило на Public.
